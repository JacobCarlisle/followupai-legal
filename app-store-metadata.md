## App Store Connect checklist

1. Upload the site files to the root of `JacobCarlisle/followupai-legal`.
2. Enable GitHub Pages from the `main` branch and `/root` folder.
3. Support email is set to `jacobrcarlisle@gmail.com`.
4. Open each URL in a private/incognito browser and confirm it returns a public page with no login requirement.
5. In App Store Connect, use `privacy.html` as the required Privacy Policy URL.
6. Use `support.html` as the Support URL.
7. Use `privacy-choices.html` as the optional User Privacy Choices URL if you want the App Store product page to include a privacy choices link.
8. Complete App Privacy answers based on the app’s real behavior, including data processed by any third-party AI, analytics, crash reporting, authentication, cloud storage, or integration providers.

## Suggested App Privacy categories to review

Do not copy this blindly into App Store Connect. Use it as a checklist against the production app:

- Contact Info, such as email address, if accounts or support requests are collected.
- User Content, such as notes, prompts, drafts, reminders, tasks, or messages entered in the app.
- Contacts or Calendar data, if the app accesses those permissions or integrations.
- Identifiers, such as user ID or device ID, if used by authentication, analytics, or crash reporting tools.
- Usage Data, if analytics are collected.
- Diagnostics, if crash logs or performance data are collected.
- Purchases, if subscriptions or in-app purchases are offered.

Apple expects App Privacy disclosures to include data collected by the developer and by third-party partners integrated into the app.
